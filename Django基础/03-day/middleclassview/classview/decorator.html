类视图使用装饰器

为类视图添加装饰器，可以使用三种方法。

为了理解方便，我们先来定义一个为函数视图准备的装饰器（在设计装饰器时基本都以函数视图作为考虑的被装饰对象），及一个要被装饰的类视图。

def login_require(func_view):

    def wrapper(request，*args,**kwargs):
        if False:
            return func_view(request,*args,**kwargs)
        else:
            return HttpResponse("您没有登陆")
    return wrapper


class CenterView(View):

    def get(self,request):
        return HttpResponse("OK")

    def post(self,request):
        return HttpResponse("OK")

4.1 在URL配置中装饰 

from book.views import login_require
urlpatterns = [

    url(r'^center/$',login_require(CenterView.as_view()),name='center'),
]

此种方式最简单，但因装饰行为被放置到了url配置中，单看视图的时候无法知道此视图还被添加了装饰器，不利于代码的完整性，不建议使用。

此种方式会为类视图中的所有请求方法都加上装饰器行为（因为是在视图入口处，分发请求方式前）。

4.2 在类视图中装饰

在类视图中使用为函数视图准备的装饰器时，不能直接添加装饰器，需要使用method_decorator将其转换为适用于类视图的装饰器。

装饰文档

methoddecorator装饰器还支持使用name参数指明被装饰的方法。method_decorator

@method_decorator(login_require,name='dispatch')
class CenterView(View):

    def get(self,request):
        return HttpResponse("OK")

    def post(self,request):
        return HttpResponse("OK")

4.3 构造Mixin扩展类

使用面向对象多继承的特性。

class LoginRequireMixin(object):
    @classmethod
    def as_view(cls,**kwargs):
        view = super().as_view(**kwargs)
        view = login_require(view)
        return view

class CenterView(LoginRequireMixin,View):

    def get(self,request):
        return HttpResponse("OK")

    def post(self,request):
        return HttpResponse("OK")

使用Mixin扩展类，也会为类视图的所有请求方法都添加装饰行为。
